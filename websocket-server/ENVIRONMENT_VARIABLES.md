# Environment Variable Management (`websocket-server`)

This document outlines the environment variables used to configure the `websocket-server` application. These variables allow for flexible deployment and customization without modifying the codebase.

## Configuration Hierarchy

The application loads configuration values in the following order of precedence:

1.  **Environment Variables**: Values set in the environment (e.g., via a `.env` file or system environment variables) take the highest precedence.
2.  **`config/default.json`**: If an environment variable is not set, the application attempts to use the corresponding value from the `config/default.json` file.
3.  **Hardcoded Defaults**: In some cases, if a value is not found in environment variables or `config/default.json`, a hardcoded default within the application code might be used (though this is less common for primary configurations).

It is recommended to manage most configurations using a `.env` file in the `websocket-server` directory, created by copying and modifying the `.env.example` file.
